## الهيكلية العامة

بولونا هو نطام متكامل لادارة الجامعات ولتسهيل استخدام نظام بولونا في الجامعات

عند إنشاء مشروع Docus جديد باستخدام `npx create-docus my-docs`، ستحصل على:

```bash
my-docs/
├── content/             # محتوى Markdown الخاص بك
│   ├── index.md         # الصفحة الرئيسية
│   └── docs/            # صفحات التوثيق
├── public/              # الأصول الثابتة
└── package.json         # التبعيات والسكربتات
```

لا يزال بإمكانك استخدام أي ميزة أو ملف من مشروع Nuxt الكلاسيكي:

```bash
my-docs/
├── nuxt.config.ts       # تكوين Nuxt (إضافة وحدات إضافية، مكونات، إلخ)
├── app/                 # مجلد التطبيق
    ├── app.config.ts    # تكوين التطبيق
│   ├── components/      # المكونات (أضف مكوناتك الخاصة)
│   ├── layouts/         # التخطيطات (أضف تخطيطاتك الخاصة)
│   └── pages/           # الصفحات (أضف صفحاتك الخاصة)
└── server/              # كود الخادم (أضف كود الخادم الخاص بك)
```

### مجلد `content/`

هنا تكتب الصفحات باستخدام Markdown. يقوم Docus تلقائيًا بإنشاء مسارات بناءً على هيكل ملفاتك.

**هيكل اللغة الواحدة:**

```bash
content/
├── index.md
├── getting-started.md
└── guide/
    ├── introduction.md
    └── configuration.md
```

**Multi-language structure (with i18n):**

```bash
content/
├── en/
│   ├── index.md
│   └── guide/
│       └── introduction.md
└── fr/
    ├── index.md
    └── guide/
        └── introduction.md
```

::prose-tip
More information about i18n is available in the [Docus documentation](https://docus.dev/concepts/internationalization).
::

### `public/` directory

Files contained within the `public/` directory are served at the root and are not modified by the build process. This is where you can locate your images, icons, and other static assets.

### `package.json`

This file contains all the dependencies and scripts for your application. The `package.json` of
a Docus application is really minimal and looks like:

```json [package.json]
{
  "name": "my-docs",
  "scripts": {
    "build": "nuxt build --extends docus",
    "dev": "nuxt dev --extends docus",
  },
  "dependencies": {
    "docus": "latest",
    "better-sqlite3": "^12.2.0",
    "nuxt": "^4.0.0"
  }
}
```

### `nuxt.config.ts`

*This file is not mandatory to start a Docus application.*

You can add extra modules to your Nuxt configuration file:

```typescript [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@nuxtjs/plausible']
})
```

### `app.config.ts`

*This file is not mandatory to start a Docus application.*

::prose-warning
You need a `nuxt.config.ts` to be set if you want to override app configuration.
::

This is where you can configure Docus to fit your branding, handle SEO, set your locale, and adapt links and socials. See the [configuration documentation](https://docus.dev/concepts/configuration) for details.

```ts [app.config.ts]
export default defineAppConfig({
  docus: {
    locale: 'en', // Set your single-language locale
  },
  seo: {
    title: 'My Docs',
    description: 'My awesome documentation',
  },
  // ... other configurations
})
```

## Full Nuxt Project Capabilities

Since Docus is a Nuxt layer, you can use **any feature** of a standard Nuxt project:

::prose-warning
You need a `nuxt.config.ts` to be set if you want to override your app with Nuxt files. If no Nuxt config is created, changes will not be applied.
::

```bash
my-docs/
├── app/                 # App directory (optional)
    ├── app.config.ts    # App configuration
│   ├── components/      # Custom Vue components
│   ├── layouts/         # Custom layouts
│   ├── pages/           # Custom Vue pages (outside of content)
│   ├── composables/     # Vue composables
│   └── middleware/      # Route middleware
├── server/              # Server-side code
│   └── api/             # API routes
├── plugins/             # Nuxt plugins
├── middleware/          # Global middleware
└── modules/             # Custom Nuxt modules
```

::prose-tip{to="https://docus.dev/concepts/nuxt"}
This layer-based approach gives you the power of the entire Nuxt ecosystem while keeping documentation as the primary focus.
::
